Cours: INTRO JAVASCRIPT Le Javascript (developpeme,nt coté client - Front end developement)est tout ce qui est traité par le navigateur et affiché au visiteur (User). Nous, en tant que developpeur on est auteur (author en anglais) Le dev javascript permet principalement de reagir aux actions de l'utilisateur, directement dans le navigateur. Ajax: Asynchronous Javascript And Xml = permet d'avoir des changements sans reload de page.(ex chargement de données ou data). Les qualités requises pour être developpeur: la RIGUEUR (primordiale dans l'ecriture du code. Les navigateurs trebuchent sur tout code mal écrit) Faire tres attention et se relire constamment, valider sur le w3c, tester, tot et souvent!!! par ex, un signe ecrit deux fois au lieu d'une peut inverser totalement la signification du code! l'Organisation = Le rangement, la nomenclature, commentaires, faire du code simple, commenté et lisible) la Capacité d'abstraction= Un developpeur quand il fait face a un probleme à resoudre doit savoir fractionner, celui ci, en morceaux plus digestes et garder une vue plus globale de l'ensemble du code. Avant de commencer a écrire, il doit etre capable de s'imaginer le resultat du code qu'il va executer. La version actuelle de javascript, JS en abregé, est l'implémentation d'un langage script appellé EcmaScript. (standardisé dans les années 90 par Netscape) Inventé par Brendan Eich, confondateur du projet Mozilla. ES5 5 est utilisé depuis 2009, On arrive à ES6 aujourd'hui! JS permet: -Definir une action au clic -Verifier des données entrées dans un formulaire -Detecter des changements de page et lancer des alertes. -Stocker des valeurs utiles dans des variables -Faire des opérations sur des morceaux de texte (chaines de caracteres/ string) -Permet d'executer du code en réponse à certains évenements se produisant sur une page web -creer ou declancher des animations -faire des carousels, des menus deroulants, des galeries d'images interactives, gerer la video et l'audio -Creer des jeux, des graphiques, 2d et 3d, (Datavisualisation = graphiques, camemberts, données et chiffres sur une carte...), des applications completes fondées sur des bases de données, applis mobile (online ou offline) JS est un langae compacte, en restant souple. Les developpeurs ont écrits de nombreux outils sur le coeur de javaScript (core), creant des fonctionalités supplementaires. Parmi ces outils, il y'a les API (application Programming Interface) interface de programmation d'application pour navigateurs. 2 categories, d'API => API navigateur et API tierce particulierement Ce sont des blocs de code deja prets, qui permettent au developpement de creer plus facilement des programmes (boites a outil deja developpés et testés.) C'est comme le code en "kit" pour la programmation, Menuisier VS Ikea. Les APIs navigateur font partie integrante du navigateur, et peuvent acceder a des données de l'environnement informatique (l'ordinateur, le mobile), ou faire d'autres choses complexes. Par exemple: ****1**** -L'API DOM (Document Object Model) nous permet de manipuler du html et du css, creer , modifier, supprimer du html, appliquer de nouveaux styles a la page de façon dynamique (ex: creer une popup charger de nouveaux contenus dans une page, il s'agit de l'action du DOM.) https://fr.wikipedia.org/wiki/Document_Object_Model -API de geolocalisation: https://fr.wikipedia.org/wiki/G%C3%A9olocalisation https://developer.mozilla.org/fr/docs/Using_geolocation permet de demander a l'utilisateur ou il se trouve (coordonnées) et d'afficher ses coordonnées sur une carte. -API Canvas et webGL: permet de creer des graphiques animés en 2d(Canvas) et en 3d(wbGL): https://developer.mozilla.org/fr/docs/Web/HTML/Canvas ex Canvas: https://developer.mozilla.org/en-US/docs/Web/Demos_of_open_web_technologies https://codepen.io/soulwire/pen/Ffvlo https://codepen.io/eltonkamami/pen/ECrKd ex webGL: http://sentdex.com/1w-global-sentiment-analysis/ -API Video et Audio permet de lire de la video et de l'audio sans plugin -API web rtc: (peer to peer) https://developer.mozilla.org/fr/docs/Web/API/WebRTC_API#Connection_setup_and_management -API Webstorage: permet de faire des applications offline (cash manifest) -API Webworkers: multi thread en JS: ****2**** Les API tierces parties ne font pas parties par defaut du navigateur, on doit donc recuperer le code de la librairie (faire un lien vers la librairie) pour pouvoir l' utiliser dans son site. L'API de twitter qui permet par exemple d'afficher les derniers tweets dans votre page. https://developer.twitter.com/ L'API de ggoglemaps: permet d'afficher des cartes personalisées (api payante) https://cloud.google.com/maps-platform/ Un site ou trouver toutes les API tierce parties: https://www.programmableweb.com/category/all/apis ---------------------------------- Séparation des concepts: Regles de bases: -Pas de CSS dans Javascript - Pas de balises HTML dans le javascript -pas de Css dans le HTML (des feuilles de style interne ou externe oui mais on evite les style inline)
